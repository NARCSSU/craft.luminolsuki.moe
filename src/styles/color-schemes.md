# 高级配色方案设计文档

## 1. 优秀案例分析

### 1.1 GitHub
- **配色理念**：专业、简洁、高对比度
- **色彩组合**：深色主题以深灰为主，配以紫色主色调，强调代码的可读性
- **应用方式**：使用分层设计，不同层级使用不同深度的灰色，主色调用于强调交互元素
- **可访问性**：文本与背景对比度符合WCAG标准，确保长时间阅读舒适

### 1.2 Apple
- **配色理念**：简约、优雅、现代
- **色彩组合**：大面积留白，少量高饱和度的强调色
- **应用方式**：使用渐变和阴影创造层次感，强调色用于突出关键操作
- **可访问性**：高对比度设计，确保文本清晰可读

### 1.3 Microsoft
- **配色理念**：活力、包容、现代
- **色彩组合**：使用明亮的蓝色作为主色调，辅以绿色、黄色等强调色
- **应用方式**：采用卡片式设计，不同功能模块使用不同颜色区分
- **可访问性**：严格遵循WCAG标准，确保所有用户都能良好体验

### 1.4 Spotify
- **配色理念**：活力、动感、音乐性
- **色彩组合**：深黑色背景，配以亮绿色主色调，营造沉浸式体验
- **应用方式**：使用渐变和模糊效果创造深度感，主色调用于突出当前播放内容
- **可访问性**：文本与背景对比度符合标准，确保信息清晰可读

### 1.5 Adobe
- **配色理念**：创意、专业、多样化
- **色彩组合**：深色主题以深灰为主，配以紫色主色调，支持多种色彩主题切换
- **应用方式**：使用模块化设计，不同工具使用不同颜色标识
- **可访问性**：提供高对比度模式，满足不同用户需求

## 2. 设计思路

基于对优秀案例的分析，结合LuminolCraft网站的主题和定位，我设计了3套差异化的配色方案，均符合WCAG 2.1 AA级可访问性标准。

### 2.1 设计原则
- **一致性**：同一方案内的颜色保持协调统一
- **可访问性**：确保文本与背景对比度符合WCAG 2.1 AA级标准
- **辨识度**：不同功能模块使用不同颜色区分
- **适应性**：支持深色和浅色主题切换
- **情感表达**：根据网站主题选择合适的色彩心理学

### 2.2 色彩心理学依据
- **紫色**：代表创造力、智慧、神秘，适合科技类网站
- **蓝色**：代表信任、专业、稳定，适合企业类网站
- **绿色**：代表活力、成长、自然，适合游戏类网站
- **橙色**：代表活力、热情、友好，适合社区类网站

## 5. 实现方案

### 5.1 CSS变量结构

使用CSS变量实现配色方案，便于切换和维护：

```css
:root {
  /* 通用变量 */
  --primary-color: var(--scheme-primary);
  --secondary-color: var(--scheme-secondary);
  --accent-color: var(--scheme-accent);
  /* ...其他变量 */
}

/* 方案一：科技紫 */
[data-color-scheme="tech-purple"] {
  --scheme-primary: #8b5cf6;
  /* ...其他颜色 */
}

/* 方案二：海洋蓝 */
[data-color-scheme="ocean-blue"] {
  --scheme-primary: #3b82f6;
  /* ...其他颜色 */
}

/* 方案三：森林绿 */
[data-color-scheme="forest-green"] {
  --scheme-primary: #10b981;
  /* ...其他颜色 */
}
```

### 5.2 配色切换功能

实现一个配色切换组件，允许用户在不同方案间切换：

```vue
<template>
  <div class="color-scheme-switcher">
    <button 
      v-for="scheme in colorSchemes" 
      :key="scheme.id"
      :class="{ active: currentScheme === scheme.id }"
      @click="switchScheme(scheme.id)"
      :style="{ backgroundColor: scheme.primary }"
    >
      {{ scheme.name }}
    </button>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue';

const colorSchemes = [
  { id: 'tech-purple', name: '科技紫', primary: '#8b5cf6' },
  { id: 'ocean-blue', name: '海洋蓝', primary: '#3b82f6' },
  { id: 'forest-green', name: '森林绿', primary: '#10b981' }
];

const currentScheme = ref(localStorage.getItem('colorScheme') || 'tech-purple');

const switchScheme = (schemeId: string) => {
  currentScheme.value = schemeId;
  document.documentElement.setAttribute('data-color-scheme', schemeId);
  localStorage.setItem('colorScheme', schemeId);
};

onMounted(() => {
  document.documentElement.setAttribute('data-color-scheme', currentScheme.value);
});
</script>
```

- 文本与背景对比度符合要求

- **与GitHub对比**：同样采用深色主题，但使用更丰富的强调色

- **与Microsoft对比**：同样支持多种配色方案，但更注重主题的情感表达
- **与Spotify对比**：同样采用沉浸式设计，但更注重信息的可读性
- **与Adobe对比**：同样支持主题切换，但更注重主题的差异化
